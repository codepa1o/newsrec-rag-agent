# AI 新闻推荐系统治理测试资料

## 推荐透明度

新闻推荐系统需要向用户说明主要推荐依据，包括用户近期点击、收藏、浏览历史、内容类别偏好以及关键词匹配情况。透明的推荐理由可以帮助用户理解为什么看到某篇新闻，也能减少用户对个性化推荐的误解。

## 可解释推荐

在新闻推荐场景中，可解释性不仅是展示一句推荐理由，还应该说明证据来源。例如，系统可以引用用户上传的行业报告、政策资料或研究论文，说明某篇新闻和用户长期关注主题之间的关系。

## 反馈闭环

推荐系统应该支持喜欢、不感兴趣、屏蔽类别、收藏和浏览历史等反馈信号。系统需要把这些反馈写入用户画像，并在后续召回和重排序阶段动态调整推荐结果。

## 幻觉控制

当本地资料库没有足够证据时，RAG 系统应该明确说明“本地资料中没有找到足够依据”，而不是编造不存在的政策、数据或结论。回答必须尽量引用检索到的资料片段。

## 评估指标

新闻推荐系统可以使用 HitRate@K、MRR@K 和 NDCG@K 评估排序效果。RAG 问答模块还可以评估 faithfulness、answer relevance 和 citation coverage。
